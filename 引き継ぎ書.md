## 1. システム概要

毎朝8:00にWindows Task Schedulerで自動起動し、51キーワードのニュースをGoogle News RSSから収集→HTMLレポート生成→GitHub Pagesにデプロイする情報収集システム。

### 処理フロー

```
[Task Scheduler 08:00]
       ↓
run_collection.bat
  ├─ Step1: Context.py
  │    ├─ 51キーワード × Google News RSS取得（各最大20件）
  │    ├─ キーワード別Markdownファイル出力（outputs/キーワード-YYYY-MM-DD.md）
  │    ├─ URL・タイトル類似度による重複排除
  │    └─ キーワード別→日次統合HTMLレポート生成（reports/レポート-YYYY-MM-DD.html）
  └─ Step2: publish_to_github.py
       ├─ reports/ → docs/ へコピー（ASCIIファイル名に変換）
       ├─ インデックスページ（context-reports.html）再生成
       └─ git add/commit/push → GitHub Pages に反映
```

## 5. Windows移行時に実施した主要修正

### 5-1. Task Scheduler環境でのパッケージ読み込み問題
**原因**: Task SchedulerはAPPDATA環境変数を持たないため、ユーザーsite-packagesを参照できない  
**対処**: Context.py冒頭でsys.pathを直接ハードコード

```python
# Context.py 先頭（絶対に削除しないこと）
import sys, os
_base = r"C:\Users\hisas\AppData\Roaming\Python\Python314\site-packages"
for _p in [_base,
           os.path.join(_base, "win32"),
           os.path.join(_base, "win32", "lib"),
           os.path.join(_base, "Pythonwin")]:
    if _p not in sys.path:
        sys.path.insert(0, _p)
```

### 5-2. SSL証明書エラー
**原因**: WindowsのPythonはシステム証明書ストアを使わない  
**対処**: ContentCollector初期化時に `session.verify = False` + warnings抑制

```python
import urllib3
urllib3.disable_warnings(urllib3.exceptions.InsecureRequestWarning)
# ContentCollector.__init__ 内
self.session.verify = False
```

### 5-3. lxml未インストールによる収集0件
**原因**: BeautifulSoupのXMLパーサー(`'xml'`)はlxmlが必須。未インストールで全件スキップ  
**対処**: `pip install lxml` 済み（setup_task_scheduler.ps1に含める）

### 5-4. git push 非fast-forward
**対処**: publish_to_github.py で push前に `git fetch origin` を実行、`--force-with-lease` でプッシュ

## 9. 既知の制限・注意事項

1. **last_check は収集結果が0件でも更新される**  
   収集中にエラーが起きても `config.json` の `last_check` が更新される。  
   翌日になるまで再収集されない。手動リセット方法：
   ```powershell
   # config.jsonを開いてlast_checkの対象キーワードの日時を過去日に書き換えるか削除
   notepad "C:\Users\hisas\KAIHATU\コンテキスト\config.json"
   ```

2. **DuckDuckGo検索は現在無効**  
   ボット検知によりブロックされるためコメントアウト済み（`collect_all()`内）。  
   現在はGoogle News RSSのみ使用。

3. **Reddit検索は英語キーワードのみ有効**  
   `collect_all()` で `'en' in languages` の場合のみ呼び出し。  
   config.jsonの `language` が `"ja"` のため現在は無効。

4. **Flask Web UIは現在未活用**  
   `http://127.0.0.1:5000` でローカルWeb UIが起動するが、通常の自動実行（バッチ起動）では使わない。

5. **Gemini API キーが config.json に平文保存**  
   `config.json` にGemini APIキーが記載されている（`ai_enabled: true`）。  
   GitHubへはdocs/のみpushしており、config.jsonはpushしていないが取り扱いに注意。
